# Final Release Notes

For ‘release’ commits, also verify the release note makes it into the [https://admin.sparkdevnetwork.org/CoreDeveloperReleaseNotes](https://admin.sparkdevnetwork.org/CoreDeveloperReleaseNotes) correctly. For example, if a *new* feature in an upcoming major release was fixed during the pre-alpha release (of that upcoming major release), make sure the Release column is not filled in for that fix as illustrated here:
