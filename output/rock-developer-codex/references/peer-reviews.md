# Peer Reviews

Spark conducts Peer Reviews to:

-   Familiarize developers with code written by other members of the team.
-   Improve our shared knowledge and implementation of standard patterns by getting feedback for things you might not be aware of.
-   Facilitate collaborative recommendations for alternate approaches (that fit our patterns/approaches).
-   Provide a backstop to the original developer.
