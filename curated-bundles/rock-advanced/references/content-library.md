# Library Viewer

The Library Viewer is an intuitive gateway to a wide array of articles and resources. Seamlessly search by topic, date, and popularity, downloading content into your Rock instance with ease. Whether enhancing connections or tackling challenges, the library's transformative wisdom enriches minds and hearts, all while aligning seamlessly with your digital strategy.

You’ll use a Content Channel to access the Library Viewer, as described below.

Once in the Library Viewer you’ll see a list of content items along the left. You can click on any of these to see the associated content. You’ll also notice a “Download Article” button, which will download the content as a new Content Channel Item into the Content Channel from which you accessed the library.

![Content Library Viewer](https://rockrms.blob.core.windows.net/documentation/Books/14/1.18.0/images/content-library-viewer-v18.png)

Content Library Viewer

If you’ve already downloaded the item, the “Download Article” button will change to say “Re-download and Refresh” which will update (overwrite) the item from your previous download. If you made any changes to the content after you downloaded it, those changes will be lost.

You may also notice a green icon that appears both within the item itself and in the list of items along the left. This is simply a visual indicator that the item has been downloaded.

# Setup

The Content Library features are enabled on specific Content Channels (Admin Tools \> CMS Configuration \> Content Channels) and apply to all the items within the channel. You can then choose individual items to share or download.

![Content Channel Setup](https://rockrms.blob.core.windows.net/documentation/Books/14/1.18.0/images/content-library-content-channel-setup-v18.png)

Content Channel Setup

As mentioned above, there are three License options available to you:

1. **Open:** Enjoy ultimate flexibility—use articles as needed, without attribution. Customize downloaded content to suit your organization.
2. **Author Attribution:** Use articles as-is, with no changes. Acknowledge the original author's contribution when showcasing the article.
3. **Organization Attribution:** Use articles unaltered. Attribute both the author (if specified) and the organization that added it to the library when displaying the article.

With an Open License you won’t want to give attribution to the author, because as soon as you change any of the content it’s no longer their work. If you change the License, any items you’ve previously uploaded to the library will retain their original License. So, changing the License only impacts new uploads going forward.
